Custom Transformer From Scratch

Deep Learning LLM PyTorch

Méthodologie et Architecture

Ce projet vise à implémenter un modèle de langage (LLM) de zéro, sans dépendre de bibliothèques haut niveau comme HuggingFace, en se basant uniquement sur PyTorch. L'objectif est de maîtriser les mécanismes internes des Transformers modernes.

J'ai intégré plusieurs optimisations clés, notamment :

  • Grouped-Query Attention (GQA) : Pour réduire la taille du cache KV (Key-Value) en mémoire lors de l'inférence.
  • Rotary Position Embedding (RoPE) : Pour une meilleure généralisation sur des longueurs de séquence variables par rapport aux encodages positionnels absolus.

Structure du Checkpoint

Au cours de l'entraînement, je sauvegarde régulièrement l'état du modèle et de l'optimiseur dans des fichiers .pt. Voici un aperçu de la structure du dictionnaire sauvegardé :

checkpoint = { 'model': raw_model.state_dict(), 'optimizer': optimizer.state_dict(), 'iter_num': iter, 'best_val_loss': best_val_loss, 'config': config } torch.save(checkpoint, checkpoint_path)

Prochaines Étapes

Une fois l'entraînement terminé, je prévois de charger les poids directement dans le backend de ce site. Ainsi, la section "Mon Modèle LLM" du portfolio sera directement connectée à ce modèle via une API locale, ce qui permettra de tester l'inférence en temps réel !
